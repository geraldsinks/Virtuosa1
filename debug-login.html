How to Test

Important: Open browser developer tools (F12) and go to Console tab to see debug messages.
  1. Open Login Page
  2. Open browser developer tools (F12) → Console tab
  3. Try to login with email and password
  4. Look for debug messages in console
  5. Check what data is being sent and received

Possible Issues to Check

1. Form Element IDs

Check if the form elements have correct IDs:

  • Email input: id="login-email"
  • Password input: id="login-password-input"

2. Network Requests

In developer tools Network tab, check:

  • Request URL: Should be https://api.virtuosazm.com/api/auth/login
  • Request Method: POST
  • Content-Type: application/json
  • Request Body: Should contain email and password

3. Server Response

Check the server response:

  • Status Code: 400 indicates client-side issue
  • Response Body: Should contain error message

Common Issues & Solutions

If you see "Email and password are required":
  • Form elements might not have correct IDs
  • JavaScript might not be getting values correctly
  • Request body might be empty or malformed
If you see network/CORS errors:
  • API endpoint might be incorrect
  • Server might not be running
  • CORS configuration issue
If you see validation errors:
  • Email or password format issues
  • Missing required fields
  • Database connection issues
